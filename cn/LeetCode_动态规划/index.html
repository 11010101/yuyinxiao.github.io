动态规划

基于一维、二维数组，实现动态规划思路。区别于贪心，动态规划的每一个状态是由上几个状态决定的。贪心没有状态依赖，是局部选最优至全局最优。部分问题既可以用贪心解决，也可以用DP，相比较此时贪心一般节省空间复杂度，但DP递推公式的形式更能显式的说明问题。

  1. 确定dp数组及下标的含义
  2. 确定dp的递推公式
  3. 如何初始化、与遍历规则
  4. 举例推导，验证规则
  • 判断是否可以转换为DP问题
  • 如何确定递推公式等

分类

  1. 求最优解类，典型问题是背包问题，最优子结构：即当前问题的最优解取决于子问题的最优解；
  2. 累积计数类；存在一定的递推性质。当前问题的方案数取决于子问题的方案数。

背包问题

0-1背包、完全背包、多重背包

参考-滚动数组

  1. 不同阶段，递进的进行状态转移，当前阶段的状态，依赖于上一阶段；
  2. 滚动数组中，因为覆盖不可顺序遍历，因此同一阶段的状态不可横向转移；
  3. 递推公式决定了递推的方向；初始化决定了起点是否正确；滚动数组的调整，完全是因为覆盖导致的不同阶段的状态前后共存

多重背包：N种物品和一个容量为V的背包。第i种物品最多有M_i件可用，每件耗费的空间是Ci ，价值是Wi。将哪些物品装入背包可使这些物品的耗费的空间 总和不超过背包容量，且价值总和最大。（出现很少）

0-1背包问题

  • 0-1背包和完全背包的区别：有限抽取和无限抽取；纯完全背包的遍历顺序先后可以替换；理论上，01背包的原问题扩展为可重复，则是完全背包，递推上转为累加
  • 0-1背包和完全背包，都有一维和二维的实现。

有N件物品，和重量为W的背包。第i件物品的重量是weight[i]，得到的价值是value[i]。将哪些物品装包，价值总和最大。

二维DP背包

  1. 确定dp[i][j]：表示从0~i的物品里任取，放入容量为j的背包的价值总和；
  2. 确定递推公式dp[i][j] = max( dp[i-1][j], dp[i-1][j-weight[i]] + value[j])；
    1. dp[i-1][j]，代表当前背包重量为j，不放入i物品时的价值；
    2. dp[i-1][j - weight[i]] + value[j]，代表当前背包放入物品i后，之前重量为j - weight[i]时的价值，另加上物品i的价值。
  3. dp数组初始化
    1. dp[i][0]，代表背包余量为0，价值为0；
    2. dp[0][j]，代表放入物品i0，各重量j的价值，j < weight[0]时，dp[0][j]价值为0，因为不够放入物品i0，j >= weight[0]，dp[0][j]价值均为value[0]；

滚动数组实现（重要）

滚动数组：需要满足的条件是上一层可以重复利用，直接拷⻉到当前层，把二维dp降为一维dp。如何确定何种情况，上一层可以重复利用？

  1. 更新dp[j]的含义：代表背包重量为j时，可取物品的最大价值；
  2. 更新递推公式：dp[j] = max(dp[j], dp[j - weight[i]] + value[i]);，放物品i，和不放物品i，取价值最大；
  3. 初始化与遍历
    1. 二维dp遍历，背包容量从小到大；
    2. 一维dp遍历，背包从大到小，因为顺序遍历，会导致重复抽取；

0-1背包：当前物品i，面对所有包可能的情况，选择加到此包还是不加。

1049. 最后一块石头的重量II

石头粉碎，使剩下的最小。重量相近的进行粉碎，剩下较小的差，再继续粉碎。实质上是将石头分成重量相近的两堆，保证相撞之后剩下的石头最小，转为01背包问题。

  1. dp[j]定义为，容量为j=sum/2的背包，最多盛dp[j]重的石头。
  2. 递推公式为dp[j] = max(dp[j], dp[j - stones[i]] + stones[i])

416. 分割等和子集

熟悉0-1背包问题，此处就是选择几个数，要求其和为sum(nums)/2。看透0-1背包的本质，其他就只是分析清楚，套用模版即可。

二维DP

  1. dp[i][j]：表示从数组的[0,i]下标范围内选取正整数（可以是 0 个），是否存在方案使得正整数和等于j；
  2. 递归公式：
    1. 当j >= nums[i]，取或不取只有一个为true即可
      1. 如果不选取 nums[i]，则dp[i][j] = dp[i−1][j]
      2. 如果选取 nums[i]，则 dp[i][j] = dp[i−1][j−nums[i]]
    2. 当j < nums[i]时，当前和小，不可选取（背包小，不能装下该nums[i]）则dp[i][j] = dp[i-1][j]
  3. 初始化：
    1. 首行：dp[0][j] = false，意为仅选择第一个和不能为j；但dp[0][nums[0]] = true，选择第一个元素组成和为nums[0]
    2. 首列：dp[i][0] = true，意为和为0，当前所有都不取时

滚动数组

很明显的0-1背包，转换为滚动数组

  1. dp[i]定义为：当和为j时，能否完成分割；
  2. 递推公式为：dp[j] = dp[j] || dp[j - nums[i]]
  3. 初始化dp[0]=true

474. 1和0

给定二进制字符串数组和两个整数m和n，返回回strs的最大子集的大小。该子集中最多有m个0和n个1。
两个维度的0-1背包，即要装0，还要装1

  1. 定义dp[i][j]，为最多有i个0和j个1的strs的最大子集的大小。
  2. 递推公式dp[i][j] = max(dp[i][j], dp[i - zeroNum][j - oneNum] + 1);

322. I（组合数）

1,1,2和1,2,1是一种。

  1. 定义dp[j]，为凑足总额j时所需钱币的最少个数。
  2. 递推公式：dp[j] = min(dp[j-coins[i]] + 1, dp[j])
  3. 增加额外的判断。

518. II（排列数）

1,1,2和1,2,1不是一种，重复叠加。

  1. 定义dp[j]，为凑的j时，所需的钱币总数
  2. 递推公式：dp[j] += dp[j - coins[i]]，区别于组合数，排列数是之前种类的叠加

377. 组合总和IV

给定数组，寻找组合目标和的总排列数。

  1. 定义dp[i]，凑成目标和为i的排列个数。
  2. 递推公式：dp[i] += dp[i - nums[j]]
  3. 增加额外的条件判断

279. 完全平方数

给定n，返回和为n的完全平方数的最少数量。

  1. 定义dp[i]，为将i按照平方数分割的最小个数
  2. 递推公式为：dp[i] = min(dp[i], dp[i - j * j] + 1)，最少完全平方数组合的累加。

139. 单词拆分

给定词表和字符串，判断是否能划分为词表的单词。

  1. 定义dp[i]，为当前长度的字符串子串是否能，恰好划分。
  2. 递推公式为：j前部分子串为true && 后半部分为单词，则更新为true。
